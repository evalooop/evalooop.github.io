Understanding the ASL Metric and EVALOOP Evaluation Framework

Average number of Sustainable Loops (ASL) Metric

ASL (Average number of Sustainable Loops) measures LLM robustness in programming through self-consistency evaluation using iterative code generation and summarization loops.

Key Innovation

EVALOOP exploits the natural duality between complementary software engineering tasks (code generation and code summarization) within a self-contained feedback loop, providing a unified metric for robustness without relying on external adversarial attacks.

Definition

The ASL metric quantifies model robustness by measuring the average number of sustainable loops before failure in the code generation-summarization cycle:

ASL = Σ(last_sustainable_loop²) / (N × max_loops)

where:
- last_sustainable_loop = the last loop where the model succeeded
- N = total number of benchmark tasks
- max_loops = maximum number of loops tested (e.g., 10)
- Quadratic weighting (loop²) emphasizes later-loop performance

Key Components

Code Generation

LLM generates code from natural language specification

NL → Code
Code Summarization

LLM summarizes code back to natural language

Code → NL
Loop Iteration

Number of sustainable transformation cycles

10+ loops
Pass@1 Metric

Success rate at each iteration level

0-100%

EVALOOP Example

// Example: EVALOOP process for task "check consecutive numbers" (Max 10 loops)
Loop 1:
  Input: "Write a function to check whether given list contains consecutive numbers"
  Code: def check_consecutive(lst): return sorted(lst) == list(range(min(lst), max(lst)+1))
  Pass@1: ✓ (Test passed)

Loop 2:  
  Input: "Function to determine if list forms consecutive sequence by comparing sorted to range"
  Code: def check_consecutive(lst): return sorted(lst) == list(range(min(lst), max(lst)+1))
  Pass@1: ✓ (Test passed)

Loop 3:
  Input: "Check if list has consecutive integers from minimum to maximum value"
  Code: def check_consecutive(nums): return set(nums) == set(range(min(nums), max(nums)+1))
  Pass@1: ✓ (Test passed)

Loop 4:
  Input: "Determine if list contains unbroken sequence of consecutive integers"
  Code: def check_consecutive(lst): return set(lst) == set(range(min(lst), max(lst)+1))
  Pass@1: ✗ (Test failed - handles duplicates incorrectly)

Loops 5-10: Not reached (model failed at Loop 4)

ASL contribution for this single task:
Sustainable loops = 3 (failed at Loop 4)
ASL = 3² / 10 = 9 / 10 = 0.9

For full ASL score across N tasks:
ASL = Σ(last_sustainable_loop²) / (N × max_loops)

EVALOOP Evaluation Framework

EVALOOP exploits the natural duality between code generation and code summarization tasks to create a self-contained feedback loop for robustness evaluation.

EVALOOP Process

Initial Code Generation

Given a natural language specification, the LLM generates code solution

Code Summarization

The generated code is summarized by the same LLM into a new specification

Loop Iteration

Use the new specification to generate code again, creating a feedback loop

Functional Testing

Test generated code against original test cases to verify correctness

Loop Continuation

Continue the loop until generated code fails functional tests

ASL Calculation

Compute the average number of sustainable loops before failure

Evaluation Benchmark

MBPP+ (Mostly Basic Python Problems Plus)

EVALOOP uses the MBPP+ benchmark, an enhanced version of the original MBPP dataset with additional test cases for more rigorous evaluation.

Dataset Statistics:
  • 378 programming tasks
  • Python function synthesis
  • Natural language descriptions
  • Multiple test assertions per task
Evaluation Setup:
  • Maximum 10 loops per task
  • Pass@1 metric at each iteration
  • Quadratic weighting (i²) for ASL
  • Self-contained feedback loop

Evaluation Metrics

Metric Formula Description
Pass@1 Success / Total Percentage of tasks passing all test cases in each loop
Loop Count Max sustainable iterations Number of successful transformations before failure
ASL Score Σ(i² × Pass@1_i) / N Weighted average emphasizing sustained performance
Degradation Rate ΔPass@1 / Loops Performance drop across transformation cycles

Evaluation Algorithm

The EVALOOP algorithm orchestrates the entire evaluation pipeline from test generation to final score calculation.

Algorithm Pseudocode

Algorithm: EVALOOP(model, benchmark)
────────────────────────────────────────────────
Input:  model - LLM to evaluate
        benchmark - Collection of programming tasks (e.g., MBPP+)
Output: ASL - Average number of Sustainable Loops

1: Initialize total_score = 0
2: Initialize max_loops = 10

3: for each task in benchmark:
4:     loop_count = 0
5:     current_prompt = task.description
6:     test_cases = task.test_cases
7:     
8:     for i in range(1, max_loops + 1):
9:         // Step 1: Generate code from current prompt
10:        generated_code = model.generate_code(current_prompt)
11:        
12:        // Step 2: Test the generated code
13:        test_result = execute_tests(generated_code, test_cases)
14:        
15:        if test_result == PASS:
16:            loop_count = i
17:            
18:            // Step 3: Summarize code to create new prompt
19:            current_prompt = model.summarize_code(generated_code)
20:            current_prompt += "\n" + task.assertion  // Keep test assertion
21:        else:
22:            break  // Exit loop on first failure
23:    
24:    // Calculate weighted score for this task
25:    task_score = 0
26:    for j in range(1, loop_count + 1):
27:        task_score += j * j  // Quadratic weighting
28:    
29:    total_score += task_score

30: // Calculate final ASL score
31: ASL = total_score / len(benchmark)

32: return ASL

Key Functions

function generate_code(prompt: string) → string

Generates Python code solution from natural language specification.

  • Takes task description as input
  • Returns complete Python function implementation
  • Maintains consistency with original test assertions

function summarize_code(code: string) → string

Generates natural language description from Python code.

  • Creates new task specification from code implementation
  • Preserves functional intent while varying description
  • Forms the basis for next loop iteration

function execute_tests(code: string, test_cases: array) → boolean

Validates generated code against original test assertions.

  • Executes code with provided test cases
  • Returns PASS if all tests succeed
  • Returns FAIL on first test failure

Key Insights

Performance Degradation: 5.01% - 19.31% drop in pass@1 within 10 loops

Key Findings:
- Robustness ≠ Initial Performance
- Qwen2.5-Coder-32B shows superior sustained performance
- Different models exhibit distinct degradation patterns
- Quadratic weighting better captures long-term stability

Metric Comparison

How ASL compares to other evaluation metrics in the field.

Metric Description Advantages Limitations
EVALOOP (Ours) Self-consistency evaluation through code generation-summarization loops
  • No external attacks needed
  • Unified metric across models
  • Reveals true robustness patterns
  • Configuration-independent
  • Requires dual capabilities
  • Multiple API calls per task
Pass@k Probability of passing with k attempts
  • Simple to understand
  • Widely adopted
  • Binary pass/fail only
  • No difficulty consideration
  • Limited robustness insight
BLEU Score N-gram overlap with reference solutions
  • Language agnostic
  • Fast computation
  • Doesn't test functionality
  • Style-dependent
  • Multiple valid solutions
Adversarial Attacks Perturb prompts at character/word/sentence levels
  • Direct robustness testing
  • Multiple attack types
  • Configuration-dependent
  • May favor certain models
  • Inconsistent results
  • External setup required
